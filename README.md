## Requirements

PHP >= 8.4

## Key features

- PHP LSParrot Engine AST support via your PHP CLI runtime
- LSParrot Engine-only LSP server entrypoint via `LSParrot\start_lsp()`; no PHP library runtime is required
- Composer class discovery from classmap, PSR-4, and PSR-0 autoload metadata
- PHPDoc type, template, generic, and array-shape completions
- Optional PHPStan and/or Psalm diagnostics when project-local tools are installed
- LSParrot Engine symbol index for fast project and workspace symbol lookup
- Serialized lsparrot project index cache in `.lsparrot/lsparrot-index.bin` for faster restarts

## Usage

```bash
$ pie install zeriyoshi/ext-lsparrot
$ composer require --dev phpstan/phpstan // optional
$ composer require --dev vimeo/psalm // optional
$ php -dextension=lsparrot -r 'LSParrot\start_lsp(["analyzer" => "lsparrot"]);'
```

The lsparrot Composer symbol index is cached under `.lsparrot/` and reused when
the PHP/composer metadata fingerprint still matches. LSP-managed PHPStan and
Psalm runs also place their tool caches under `.lsparrot/phpstan/cache` and
`.lsparrot/psalm/cache`.

Set `LSPARROT_NO_INDEX=1` to disable loading and saving the serialized lsparrot
index cache at `.lsparrot/lsparrot-index.bin`. This is useful for cache-behavior
tests and for forcing a fresh lsparrot Composer symbol scan on every start.
